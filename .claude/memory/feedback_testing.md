Toujours écrire des tests d'intégration — ne jamais utiliser de mocks.

**Why:** Préférence explicite de l'utilisateur.

**How to apply:** Pour tout code de test, privilégier systématiquement les tests d'intégration réels. Ne jamais proposer ou écrire de mocks, stubs ou spies comme solution de test.
